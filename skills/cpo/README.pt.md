# Chief Product Officer AI

## O que faz

Orquestra ciclos completos de produtos, desde a concepção até a produção. Transforma ideias em aplicações implementadas através de descoberta estruturada, planejamento estratégico, implementação iterativa e validação rigorosa. Funciona em cinco fases: Discovery (qualificação da ideia), Planning (estratégia e decomposição em epics/stages), Implementation (desenvolvimento stage-by-stage com testes), Validation (testes integrados) e Delivery (documentação e deploy).

Coordena seis agentes especializados (Pesquisa de Produto, CTO Advisor, Frontend Design, Backend API, Database Setup, Deployment) que trabalham em paralelo ou sequencialmente conforme a estratégia de execução. Utiliza Memory MCP para cachear pesquisas e aprender padrões entre projetos.

## Como invocar

```bash
/cpo-go <nome-projeto> <descrição>
```

**Exemplos:**

- `/cpo-go jogo criar um jogo interativo da velha`
- `/cpo-go taskflow construir app de gestão de tarefas para pequenos times`
- `/cpo-go artmarket criar marketplace para artistas venderem arte digital`

## Quando usar

- **Validar ideias de produtos** antes de investir em desenvolvimento
- **Planejar produtos complexos** com múltiplas áreas (frontend, API, banco de dados)
- **Executar produtos de forma sistemática** com qualidade em cada etapa
- **Documentar e fazer deploy** de aplicações prontas para produção
