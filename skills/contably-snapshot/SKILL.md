# Contably Codebase Snapshot

Generates a `CODEBASE-REFERENCE.md` in the Contably repo root that serves as a pre-computed context snapshot for all future Claude Code sessions. This eliminates repeated codebase discovery and gives subagents instant context.

## When This Runs

- **On demand:** User invokes `/contably-snapshot`
- **Post-deploy:** Auto-invoked after successful `/deploy-conta-staging` or `/deploy-conta-production`
- **Periodically:** User can schedule via `/loop` or `/schedule` if desired

## Output Location

`/Volumes/AI/Code/contably/CODEBASE-REFERENCE.md`

Referenced by `/Volumes/AI/Code/contably/CLAUDE.md` so it auto-loads every session.

## Workflow

### Step 1: Gather Raw Data (Parallel)

Launch 4 parallel Explore agents (model: haiku) to collect data simultaneously:

**Agent 1 — Tech Stack & Dependencies:**

- Read `package.json` (root), `pyproject.toml`, `apps/api/requirements*.txt`
- Read `apps/admin/package.json`, `apps/client-portal/package.json`, `apps/mobile/package.json`
- Read `pnpm-workspace.yaml`, `turbo.json`
- Read `docker-compose.yml` for infrastructure services
- Extract: frameworks, runtimes, versions, key dependencies

**Agent 2 — API Surface & Models:**

- Glob `apps/api/src/api/routes/**/*.py` — list all route files
- For each route package, read `__init__.py` or first route file to extract endpoints
- Glob `apps/api/src/models/*.py` — list all SQLAlchemy models
- Read `apps/api/src/config/database.py` for DB config
- Extract: endpoint map (method + path + description), model list with key fields

**Agent 3 — Product Modules & Services:**

- Glob `apps/api/src/services/*.py` — list all services
- Glob `apps/api/src/workflows/**/*.py` — list all workflow/Celery tasks
- Glob `apps/api/src/integrations/**/*.py` — list all integrations
- Read key files in `apps/api/src/ai_assistant/`, `apps/api/src/reconciliation/`, `apps/api/src/validation/`
- Extract: feature map (module → purpose → key files)

**Agent 4 — Infrastructure & CI/CD:**

- Read `.github/workflows/*.yml` — all GHA workflows
- Glob `infrastructure/kubernetes/**/*.yaml` — K8s manifests
- Read `Dockerfile*` files in `apps/api/`
- Read `infrastructure/terraform/*.tf` if exists
- Read `infrastructure/monitoring/` configs
- Extract: deploy pipeline, K8s topology, monitoring setup

### Step 3: Write and Verify

1. Write `CODEBASE-REFERENCE.md` to `/Volumes/AI/Code/contably/CODEBASE-REFERENCE.md`
2. Verify the file is under 500 lines (trim if needed — this is a reference, not documentation)
3. Report: section count, total lines, key stats found

## Rules

1. **Never include secrets, API keys, passwords, or tokens** — only list env var names
2. **Never include internal IPs, Tailscale addresses, or VPS details**
3. **Keep it under 500 lines** — this loads into every session's context
4. **Use tables over prose** — scannable > readable for a reference doc
5. **Include file paths** — every claim should point to the source file
6. **Date-stamp the header** — so sessions know how fresh the snapshot is
7. **Overwrite, don't append** — each run produces a complete fresh snapshot
8. **No infrastructure credentials** — OCIR URLs are fine, OCIDs/tokens are not
